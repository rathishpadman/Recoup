# INVARIANTS.md — Recoup v2

Every invariant below is **machine-verifiable**. Each names its enforcement point and the test that proves it. **Breaking any invariant is a release blocker.** This file seeds the README claim→code→test table and is the contract the SDD, agents, and Codex build sessions must honor.

**Scope (locked via `RECONCILIATION_LEDGER.md`).** Recoup v2 delivers four agentic capabilities on a deterministic evidence spine:
**A. Closed-Loop Risk Mesh** · **B. Deduction Forensics & Recovery** (hero) · **C. Dynamic Credit Sentinel** · **D. Behavioral Containment**.
Runtime is **Node 22 + TypeScript** (modular monolith). Python is **offline-only** (synthetic data generator, design search). Old `S1–S4` labels are retained only as a crosswalk: S1→A, S2→B, S3→C, S4→D.

**Verification.** `npm run verify` = typecheck + `vitest run` + `dependency-cruiser` (port boundaries) + eval gates. CI fails on any red invariant.

**Rules for this file.** Invariants may be **added**, never weakened or silently deleted, during the build. **Retirement** is permitted **only** when the underlying feature is descoped, **only** via explicit human instruction, and **only** with a transparent retired-with-reason marker preserving the original text (see I-10, I-11). Any change requires explicit human instruction in the session brief. The numeric thresholds in I-28 and the partial-hold weights/bands in I-24 are **expert-owned constants/Day-1 tunables** defined in SDD Appendix G — Codex must read them from config and must never invent or alter them.
