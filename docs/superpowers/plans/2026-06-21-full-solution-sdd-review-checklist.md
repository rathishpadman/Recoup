## Workflow Gate

Remaining P0/P1 implementation must use the subagent-driven loop:

1. Implementer subagent owns one bounded slice and lists changed files.
2. Spec-compliance reviewer checks the actual diff against this checklist and the relevant SDD/invariant requirement.
3. Code-quality reviewer checks architecture, tests, error handling, and production risks.
4. Focused tests and final `npm.cmd run verify` are run only after reviewer findings are resolved.

Do not mark an item complete because an audit ran. Mark it complete only when the implementation exists, reviewers approve it, and verification passes.

Terminate or replace stale/slow subagents after a useful wait window; do not let persona or reviewer work block indefinitely without file movement, findings, or a concrete handoff. Current controller policy: default to a `60s` wait; extend to `120s` only when the agent is visibly active, has produced useful partials, or is running tests; then interrupt for a `30s` checkpoint, close/restart narrower or take over locally after recording root cause.
